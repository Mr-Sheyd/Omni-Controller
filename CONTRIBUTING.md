# Contributing to Omni-Controller

First of all, thank you for considering contributing to this project! 

## 🤖 A Note on Development
This project is developed with a transparent approach: I am not a professional developer and I use **AI assistance** to write and debug the code. It’s a journey of learning and creating something useful at the same time. 

If you want to help, please keep this in mind. I might not understand complex architectural jargon immediately, but I’m always open to improvements and learning!

## 🚀 How Can You Help?

### 1. Reporting Bugs
If you find a bug, please use the **Bug Report** template. Since I rely on AI and community feedback, being as specific as possible (OS version, steps to reproduce) helps a lot.

### 2. Suggesting Enhancements
Have an idea for a "Turbo mode" or better UI? Open a **Feature Request**. I’ll discuss it with my AI collaborator and see how we can implement it.

### 3. Submitting Pull Requests (PRs)
If you want to contribute code:
* **Keep it Simple**: Since I maintain this with AI, clear and well-commented code is much appreciated.
* **Explain the "Why"**: In your PR description, explain what the change does as if you're explaining it to a fellow learner.
* **Respect the Style**: Try to follow the existing naming conventions (like the `A_` prefix for widgets).
